# 🚀 一键部署说明

## 使用自动部署脚本（5分钟完成）

### 步骤1: 准备 GitHub Token

在运行脚本前，建议先准备好 GitHub Personal Access Token：

1. 访问：https://github.com/settings/tokens
2. 点击 **Generate new token** → **Generate new token (classic)**
3. **Note** 填写：`Deploy Asset Calculator`
4. **Expiration** 选择：`No expiration`（永不过期）
5. **勾选权限**：✅ `repo`（完整的仓库访问权限）
6. 点击底部绿色按钮 **Generate token**
7. **重要**：复制生成的 token（只显示一次！保存好）

### 步骤2: 运行部署脚本

#### 方法A：右键运行（推荐）
1. 找到 `deploy.ps1` 文件
2. 右键点击 → **使用 PowerShell 运行**

#### 方法B：PowerShell 运行
1. 在当前文件夹空白处，按住 `Shift` + 右键
2. 选择 **在此处打开 PowerShell 窗口**
3. 输入命令：
   ```powershell
   .\deploy.ps1
   ```

#### 如果遇到执行策略错误
如果提示"无法加载，因为在此系统上禁止运行脚本"，请：

1. 以管理员身份打开 PowerShell
2. 运行命令：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. 输入 `Y` 确认
4. 关闭 PowerShell，重新运行脚本

### 步骤3: 按提示输入信息

脚本会依次询问：

1. **GitHub 用户名**：输入你的用户名
2. **仓库名称**：输入仓库名（建议 `asset-calculator`）或直接回车使用默认
3. **是否打开浏览器创建 Token**：输入 `y` 或 `n`
4. **粘贴 Token**：粘贴你在步骤1创建的 token（输入时不显示，这是正常的）

### 步骤4: 等待完成

脚本会自动：
- ✅ 检查 Git 安装
- ✅ 创建 GitHub 仓库
- ✅ 初始化 Git 仓库
- ✅ 添加所有文件
- ✅ 提交代码
- ✅ 推送到 GitHub
- ✅ 启用 GitHub Pages
- ✅ 打开网站和设置页面

### 步骤5: 验证部署

1. 等待 1-2 分钟让 GitHub Pages 构建
2. 访问你的网站：`https://你的用户名.github.io/仓库名/`
3. 如果显示 404，再等待 1 分钟后刷新

## ❓ 常见问题

### Q: 提示 Git 未安装？
**A:** 下载安装 Git：https://git-scm.com/download/win

### Q: Token 在哪里获取？
**A:** 访问 https://github.com/settings/tokens 创建

### Q: 提示仓库已存在？
**A:** 脚本会自动使用现有仓库，或者手动删除后重新运行

### Q: 推送失败？
**A:** 检查：
1. Token 权限是否包含 `repo`
2. 网络连接是否正常
3. Token 是否正确复制（没有多余空格）

### Q: 网站显示 404？
**A:** 等待 1-2 分钟，GitHub Pages 需要构建时间

### Q: 想要更新网站内容？
**A:** 修改文件后，再次运行脚本即可
