## 🌐 GitHub Pages 部署（图文教程）

### 步骤1：注册 GitHub
1. 访问 https://github.com
2. 点击 Sign up（注册）
3. 填写邮箱、密码
4. 验证邮箱

### 步骤2：创建仓库
1. 登录后点击右上角 **+** → **New repository**
2. 填写仓库名称：`asset-calculator`
3. 选择 **Public**（公开，必须）
4. 勾选 **Add a README file**
5. 点击 **Create repository**

### 步骤3：上传文件
1. 在仓库页面点击 **Add file** → **Upload files**
2. 将以下文件拖拽到网页：
   ```
   ✅ index.html
   ✅ style.css
   ✅ calculator.js
   ✅ admin.html
   ✅ admin.js
   ✅ schemes-admin.html
   ✅ schemes-admin.js
   ✅ README.md
   ✅ 使用说明.md
   ```
3. 在底部填写提交信息：`Upload files`
4. 点击 **Commit changes**

### 步骤4：启用 Pages
1. 点击仓库顶部的 **Settings**（设置）
2. 左侧菜单找到 **Pages**
3. **Source** 选择 `main` 分支
4. 点击 **Save**
5. 等待 1-2 分钟

### 步骤5：获取网址
1. 刷新页面，顶部会显示网址
2. 格式：`https://你的用户名.github.io/asset-calculator/`
3. 🎉 打开网址即可使用！

### 步骤6：分享给他人
1. 复制网址
2. 发送给需要使用的人
3. 所有人都可以通过网址访问
4. 建议在手机上添加到主屏幕

## 📲 添加到手机桌面

### iPhone/iPad
1. 用 Safari 浏览器打开网址
2. 点击底部**分享**按钮 ⎙

5. ✅ 完成！

## 🔄 如何更新内容

### 更新文件
1. 打开 GitHub 仓库
2. 点击要修改的文件
3. 点击右上角**铅笔图标**（编辑）
4. 修改内容
5. 底部点击 **Commit changes**
6. 等待 1-2 分钟，网站自动更新

1. 点击 **Add file** → **Upload files**
2. 拖拽新文件
3. 点击 **Commit changes**

## ❓ 常见问题快速解答

### Q1: 为什么访问网址显示404？
**A:** 等待 1-2 分钟，GitHub Pages 需要时间构建。

### Q2: 手机上显示不正常？
**A:** 已经做了移动端优化，建议用最新版 Safari/Chrome 浏览器。

### Q3: 数据会不会丢失？
**A:** 数据保存在浏览器本地，除非清除浏览器数据，否则不会丢失。

### Q4: 可以设置密码吗？
**A:** 这是纯前端应用，如需密码保护，建议使用 Vercel + Cloudflare Access。

### Q5: 能用自己的域名吗？
**A:** 可以！在 GitHub Pages 设置中添加自定义域名（Custom domain）。

### Q6: 多人使用会互相影响吗？
**A:** 不会，每个用户的数据都是独立的，保存在各自的浏览器中。

## 💡 进阶技巧

### 数据备份
1. 打开浏览器开发者工具（F12）
2. 输入：`localStorage`
3. 复制所有数据到记事本保存

### 数据恢复
1. 打开开发者工具
2. 在 Console 中粘贴备份的数据
3. 刷新页面

### 批量导入方案
可以编辑 JavaScript 文件中的默认方案数据，然后重新上传到 GitHub。
