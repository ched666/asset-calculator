# 📱 移动端使用和部署指南

## 方案一：最简单 - 使用本地文件分享（推荐新手）

### 特点
- ✅ 无需服务器
- ✅ 完全免费
- ✅ 数据保存在本地浏览器
- ✅ 适合个人或小团队使用

### 使用步骤
1. 将整个 `pytools` 文件夹压缩成 ZIP 文件
2. 通过微信、QQ、钉钉等方式发送给需要使用的人
3. 对方解压后，用浏览器打开 `index.html` 即可使用
4. 手机上可以使用：
   - 微信/QQ：直接在聊天窗口打开文件
   - Safari/Chrome：下载后在文件管理器中打开

### 优点
- 即开即用，无需任何配置
- 数据私密，不上传云端

### 缺点
- 需要每次传输整个文件
- 无法实时更新

## 方案二：GitHub Pages - 免费公网部署（推荐）

### 特点
- ✅ 完全免费
- ✅ 自动获得 HTTPS 网址
- ✅ 可以随时更新
- ✅ 所有人通过网址访问

### 部署步骤

#### 1. 注册 GitHub 账号
访问 https://github.com 注册免费账号

#### 2. 创建新仓库
- 点击右上角 `+` → `New repository`
- 仓库名称输入：`asset-calculator`（或任意名称）
- 选择 `Public`（公开）
- 点击 `Create repository`

#### 3. 上传文件
- 在仓库页面点击 `uploading an existing file`
- 选择所有文件（除了 `部署指南.md`）：
  - index.html
  - style.css
  - calculator.js
  - admin.html
  - admin.js
  - schemes-admin.html
  - schemes-admin.js
  - README.md
  - 使用说明.md
- 点击 `Commit changes`

#### 4. 启用 GitHub Pages
- 进入仓库的 `Settings` → `Pages`
- Source 选择 `main` 分支
- 点击 `Save`
- 等待 1-2 分钟后，会显示网址

#### 5. 访问网址
- 网址格式：`https://你的用户名.github.io/asset-calculator/`
- 用手机扫描二维码或直接访问
- 建议添加到手机桌面快捷方式

### 优点
- 永久免费
- 访问速度快
- 可以分享给任何人
- 支持手机端完美访问

### 缺点
- 数据保存在用户浏览器本地
- 需要简单的 Git 操作（可以通过网页操作）

## 方案三：Vercel - 最快速部署（推荐进阶用户）

### 特点
- ✅ 部署最简单
- ✅ 访问速度极快
- ✅ 自动 HTTPS
- ✅ 支持自定义域名

### 部署步骤

#### 1. 访问 Vercel
打开 https://vercel.com

#### 2. 注册登录
- 使用 GitHub 账号登录（推荐）
- 或使用邮箱注册

#### 3. 导入项目
- 点击 `Add New` → `Project`
- 如果文件在 GitHub：选择对应仓库
- 如果文件在本地：
  1. 先上传到 GitHub（参考方案二）
  2. 然后在 Vercel 选择该仓库

#### 4. 部署配置
- Framework Preset: 选择 `Other`
- Root Directory: 保持默认
- 点击 `Deploy`

#### 5. 获取网址
- 部署完成后自动生成网址
- 格式：`https://项目名.vercel.app`
- 可以添加自定义域名

### 优点
- 部署速度最快（1 分钟内）
- 自动更新（推送 GitHub 后自动部署）
- 访问速度优秀
- 支持自定义域名

## 📱 移动端使用建议

### 添加到手机主屏幕

#### iPhone/iPad (Safari)
1. 在 Safari 中打开网址
2. 点击底部分享按钮 ⎙

2. 点击右上角菜单 ⋮
3. 选择"添加到主屏幕"
4. 点击"添加"

## 🔄 如何更新

### GitHub Pages 更新
1. 在 GitHub 仓库页面找到要修改的文件
2. 点击文件名 → 点击编辑按钮（铅笔图标）
3. 修改内容后点击 `Commit changes`
4. 等待 1-2 分钟自动部署完成

### Vercel 更新
- 如果项目连接了 GitHub：推送到 GitHub 后自动部署
- 如果需要重新部署：在 Vercel 项目页面点击 `Redeploy`

## 🔒 数据安全说明

### 本地存储
- 所有配置数据保存在用户浏览器的 `localStorage`
- 不会上传到服务器
- 清除浏览器数据会丢失配置

### 数据备份
建议定期备份产品配置和方案：
1. 打开后台配置页面
2. 复制产品配置或方案数据
3. 保存到记事本或云笔记

## ❓ 常见问题

### Q: 手机上能正常使用吗？
A: 完全可以！已针对移动端优化，建议添加到主屏幕使用。

### Q: 数据会丢失吗？
A: 数据保存在浏览器本地，除非清除浏览器数据，否则不会丢失。

### Q: 可以多人同时使用吗？
A: 可以。每个人的配置都是独立的，互不影响。

### Q: 需要付费吗？
A: 完全免费！GitHub Pages 和 Vercel 都提供免费方案。

### Q: 如何分享给同事？
A: 直接发送网址即可，对方用手机或电脑浏览器打开。
